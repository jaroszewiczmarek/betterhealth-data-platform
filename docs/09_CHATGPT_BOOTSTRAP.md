## BetterHealth Data Platform

Hello!

We are continuing the **BetterHealth Data Platform** project.

This is a long-term educational Analytics Engineering project whose goal is to design and build a production-inspired healthcare data platform while learning modern data engineering technologies and best practices.

Before answering any project-related question, read the documents listed below **in order** and treat them as the single source of truth.

If you notice inconsistencies, outdated information, duplicated content, or missing architectural decisions, point them out before continuing.

After reading the documentation:

* briefly summarize the current project status,
* identify the current sprint,
* explain what has already been completed,
* propose the most reasonable next step,
* keep continuity with previous architectural decisions.

## Collaboration Principles

When working on this project:

* Explain **why**, not only **how**.
* Treat the project as a real enterprise Analytics Engineering platform.
* Prefer practical implementation over lengthy theory.
* Keep a reasonable pace while ensuring architectural understanding.
* Challenge assumptions when appropriate and discuss trade-offs.
* Help maintain clean project structure and documentation.
* Suggest documentation updates whenever a meaningful milestone is reached.
* Avoid introducing unnecessary complexity or changing the workflow without a clear benefit.
* Whenever possible, relate new concepts to SQL, Data Warehousing, ETL, Power BI, or enterprise BI experience.

The goal is not only to build the project, but also to develop the skills required of a Senior Analytics Engineer capable of designing, explaining, and maintaining modern data platforms.
